## 1. 프로젝트 개요

### 1.1 시스템 개요

러닝 운동을 하는 사용자들을 위한 종합 기록 관리 웹 애플리케이션입니다. 사용자는 러닝 기록을 등록하고, 페이스를 자동으로 계산받으며, 레벨별 맞춤 목표를 추천받을 수 있습니다. 또한 실시간 날씨 정보를 확인하여 러닝 적합도를 판단할 수 있으며, 마라톤 대회 정보도 제공받을 수 있습니다.

### 1.2 주요 기능

- **회원 관리**: 일반/소셜 로그인 및 회원가입
- **실시간 날씨**: 위치 기반 날씨 정보 및 러닝 적합도 안내
- **러닝 기록**: 기록 추가/수정/삭제 및 통계 분석
- **목표 추천**: 레벨별 맞춤 목표 설정 및 진행 관리
- **마라톤 대회**: 국내/해외 대회 정보 제공
- **마이페이지**: 프로필 및 신체 정보 관리, 문의 게시판

### 1.3 사용자 유형

- **일반 사용자**: 개인의 러닝 기록을 관리하고 목표를 설정하는 사용자
- **관리자**: 문의 게시판에 답변하고 공지글을 작성하는 사용자

- 최근 기록은 최대 5개까지 표시

| 초급 | 5'30"/km 이상 | 러닝을 시작하는 단계 |

| 고급 | 4'30"/km 미만 | 숙련 러너를 위한 고강도 챌린지 단계 |

## 8. 제약 및 가정

### 8.1 기술적 제약

- 소셜 로그인은 구글과 카카오만 지원
- JWT 토큰 기반 인증 사용
- 날씨 정보는 기상청 API 사용
- 위치 정보는 브라우저 Geolocation API 사용
- 주소 변환은 Reverse Geocoding API 사용
- 지도 API는 위치 권한이 필요하며, 권한 거부 시 직접 입력만 가능

### 8.2 데이터 제약

- BMI 계산 공식: 몸무게(kg) / (키(m))²
- 페이스 계산 공식: (운동시간(초) ÷ 거리(km)) → "분:초/km" 형식
- 칼로리 계산 공식: 거리(km) × 체중(kg) × 1.036 (기본 체중 70kg 가정)
- 평균 페이스 계산 시 최근 한 달 이내의 기록만 포함
- 최근 기록은 최대 5개까지 표시
- 목표 기간은 한 달(30일) 기준

### 8.3 비즈니스 가정

- 러닝 기록이 없는 신규 사용자는 초급 레벨로 기본 설정
- 목표 달성 시 자동으로 다음 단계 목표가 제시
- 날씨 API 호출 최소화를 위해 적절한 캐싱 전략 사용
- 문의 게시판의 답변은 관리자 또는 특정 권한을 가진 사용자가 댓글로 작성
- 공지글은 관리자만 작성 가능
